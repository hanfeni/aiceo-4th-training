# korea-privacy-legal — 한국형 개인정보 컴플라이언스 진입점

## 무엇을 하는가

사용자가 **점검 대상 문서(처리방침/위탁계약/동의서) 또는 처리
기술서**를 주면, 오케스트레이터가 **단일 워크플로우로 직접 점검**
(오리지널 plugin 토폴로지 — subagent 없음)해 ① 처리 유형·항목·
흐름 파악 ② 적법근거·필수기재·국외이전 점검 ③ 개보법·PIPC 고시·
KISA 기준 대조 ④ 위험 평가 ⑤ 권고를 수행하고, 점검 또는 DPIA
초안 마크다운 1개를
`./specs/claude-for-x-kr/korea-privacy-legal/<slug>/<today>/`
에 저장한다.

이것은 Anthropic Legal의 **Privacy Legal** plugin을 기능·토폴로지·
출력 불변으로 한국에 이식한 것이다. 변형은 **3개 레이어**뿐이다 —
데이터(GDPR/CCPA·美 가이드 → 개보법·PIPC·KISA), 법체계(美·EU
프레임 → 한국 개보법 체계), 법조분류(美 privacy practice → 한국
개인정보·신용정보·정보통신망 실무).

## 언제 트리거되는가

- "개인정보 처리방침/위탁계약/동의서 점검·적법성"
- "DPIA 영향평가 초안", "개보법 위반 소지"
- "가명정보 처리 적정성", "국외이전 적법근거 봐줘"

## 트리거되지 않는 경우

- 일반 상거래 계약 → `/korea-commercial-legal`
- 근로계약·노무 분쟁 → `/korea-employment-legal`
- 자영업자 1차 계약 훑기 → `/korea-contract-reviewer`
- 본격 법률 자문 → 변호사 / 코드 분석 → `/reverse`

## 어떻게 동작하는가

이 스킬이 트리거되면 **`/korea-privacy-legal` 커맨드의 실행
순서를 그대로 수행**한다:

1. **Step 0** — 점검 대상 문서·기술서 수령(비면 한 번만 요청),
   Mode(A 점검 / B DPIA) 식별, 저장 경로 결정. 정보주체 식별
   정보는 산출물·로그에 마스킹.
2. **Step 1** — 오케스트레이터가 단일 워크플로우로 직접 점검
   (처리 파악 → 적법근거·필수기재·국외이전 → 개보법·PIPC·KISA
   대조 → 위험 평가 → 권고). subagent 없음(오리지널 plugin
   토폴로지 보존).
3. **Step 2** — 마크다운 1개 저장 → grep 결정적 자가검증(변호사법
   §109 + DPIA 책임귀속 디스클레이머 + 인용 실재 검증 포함) →
   로그 append → 요약 보고.

대상이 비면 한 번만 묻고, 그 외엔 질문 없이 끝까지. 상세 계약은
`.claude/commands/korea-privacy-legal.md`를 따른다. 변호사 자문을
대체하지 않으며, DPIA 최종 책임은 개인정보처리자·CPO에 귀속됨을
강제 명시한다.
