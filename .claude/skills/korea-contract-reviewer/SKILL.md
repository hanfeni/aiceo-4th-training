# korea-contract-reviewer — 한국형 계약서 검토 진입점

## 무엇을 하는가

사용자가 **계약서 전문 텍스트**를 붙여넣으면, 오케스트레이터가
**단일 워크플로우로 직접 검토**(오리지널 Skill 토폴로지 — subagent
없음)해 ① 계약 유형·의무 파악 ② 리스크 조항 식별 ③ 한국 법·표준
약관 대조 ④ 누락 조항 식별 ⑤ 권고 정리를 수행하고, 검토 마크다운
1개를 `./specs/claude-for-x-kr/korea-contract-reviewer/<slug>/<today>/`
에 저장한다.

이것은 Anthropic Small Business의 **Contract Reviewer** Skill을
기능·토폴로지·출력 불변으로 한국에 이식한 것이다. 변형은 데이터·법
레이어(美 계약법 → 한국 민법·약관규제법·표준약관/표준계약서)와
문화·연동 레이어(DocuSign → 모두싸인 등, 단 **이 sandbox에 미연결
이라 정직히 명시**)뿐이다. 검토 자체는 텍스트만으로 완결된다.

## 언제 트리거되는가

- "이 계약서 검토/리뷰해줘", "독소조항·리스크 짚어줘"
- "근로/임대차/용역/하도급/NDA 계약서 봐줘"
- "빠진 조항 알려줘", "표준약관이랑 비교"

## 트리거되지 않는 경우

- 본격 법률 자문·소송 전략 → 변호사
- 상장사 실적·섹터 리서치 → korea-earnings-reviewer / market-researcher
- 코드베이스 분석 → `/reverse`

## 어떻게 동작하는가

이 스킬이 트리거되면 **`/korea-contract-reviewer` 커맨드의 실행
순서를 그대로 수행**한다:

1. **Step 0** — 계약서 텍스트 수령(비면 한 번만 요청), 계약 유형
   식별, 저장 경로 결정. 개인정보는 산출물·로그에 마스킹.
2. **Step 1** — 오케스트레이터가 단일 워크플로우로 직접 검토(유형
   파악 → 리스크 식별 → 한국 법·표준 대조 → 누락 식별 → 권고).
   subagent 없음(오리지널 Skill 토폴로지 보존).
3. **Step 2** — 마크다운 1개 저장 → grep 결정적 자가검증(전문가
   검토 디스클레이머 + 전자서명 미연결 명시 포함) → 로그 append →
   요약 보고.

상세 계약은 `.claude/commands/korea-contract-reviewer.md`를 따른다.
중대 사안엔 "전문가(변호사·노무사) 검토 권장"을 강제 표시하며,
법률 자문을 대체하지 않는다.
