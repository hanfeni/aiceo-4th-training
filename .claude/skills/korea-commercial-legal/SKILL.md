# korea-commercial-legal — 한국형 사내 상사 법무 검토 진입점

## 무엇을 하는가

사용자가 **계약서 전문 텍스트 + (선택) 사내 playbook**을 주면,
오케스트레이터가 **단일 워크플로우로 직접 검토**(오리지널 plugin
토폴로지 — subagent 없음)해 ① 계약 유형·당사자·핵심 의무 파악
② 강행규정·약관규제법·특별법 저촉 점검 ③ 한국 표준계약서 대조
④ 협상 라인(양보 가능/불가) 정리 ⑤ 권고를 수행하고, 검토 또는
협상 메모 마크다운 1개를
`./specs/claude-for-x-kr/korea-commercial-legal/<slug>/<today>/`
에 저장한다.

이것은 Anthropic Legal의 **Commercial Legal** plugin을 기능·
토폴로지·출력 불변으로 한국에 이식한 것이다. 변형은 **3개 레이어**
뿐이다 — 데이터(美 계약법·Westlaw → 한국 민법·상법·국가법령정보
센터), 법체계(common law 판례법 → 한국 성문법+판례), 법조분류
(美 commercial practice → 한국 상사·약관·하도급 실무).

> **기존 `korea-contract-reviewer`와의 차이:** contract-reviewer는
> Small Business 라인업(자영업자 1차 훑기). 본 하네스는 Legal
> 라인업(사내 법무팀의 강행규정·약관규제법 정밀 검토 + 협상 메모).
> 트리거는 "사내 법무 관점/강행규정/약관규제법/협상 메모"로 분리된다.

## 언제 트리거되는가

- "사내 법무 관점에서 계약 검토", "강행규정/약관규제법 위반 소지"
- "협상 메모 만들어줘", "양보 가능/불가 라인 정리"
- "표준계약서 대비 이탈 조항", "이 딜 계약 리스크 사내 검토"

## 트리거되지 않는 경우

- 자영업자 1차 계약 훑기 → `/korea-contract-reviewer`
- 개인정보 처리방침·위탁 점검 → `/korea-privacy-legal`
- 근로계약·노무 분쟁 → `/korea-employment-legal`
- 본격 법률 자문·소송 전략 → 변호사 / 코드 분석 → `/reverse`

## 어떻게 동작하는가

이 스킬이 트리거되면 **`/korea-commercial-legal` 커맨드의 실행
순서를 그대로 수행**한다:

1. **Step 0** — 계약서 텍스트·playbook 수령(비면 한 번만 요청),
   계약 유형·Mode(A 검토 / B 협상메모) 식별, 저장 경로 결정.
   개인정보·기밀은 산출물·로그에 마스킹.
2. **Step 1** — 오케스트레이터가 단일 워크플로우로 직접 검토
   (유형 파악 → 강행규정·약관규제법·특별법 점검 → 표준계약서
   대조 → 협상 라인 → 권고). subagent 없음(오리지널 plugin
   토폴로지 보존).
3. **Step 2** — 마크다운 1개 저장 → grep 결정적 자가검증(변호사법
   §109 디스클레이머 + 인용 실재 검증 포함) → 로그 append →
   요약 보고.

대상이 비면 한 번만 묻고, 그 외엔 질문 없이 끝까지. 상세 계약
(Contract·불변식)은 `.claude/commands/korea-commercial-legal.md`를
따른다. 변호사 법률 자문을 대체하지 않으며, 중대 사안엔 "변호사
검토 권장"을 강제 표시한다.
