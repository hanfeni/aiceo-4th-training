# reporter-review-scan — 기자 보도 주제군 여론 풀스캔 진입점

## 무엇을 하는가 (v2 — 실증 기반 재정의)

사용자가 **언론사명 + 기자명**(+ 선택: 기간, 기본 3개월)을 주면,
오케스트레이터가 다음을 수행한다:

1. **기사 목록 수집** sub-task를 `reporter-article-collector`에 위임
   — 기자가 기간 내 쓴 기사 URL·제목·발행일·**요약·핵심 키워드** 추출.
2. **주제 클러스터링** (오케스트레이터 본체) — N건 기사를 사건/이슈
   단위 3~6개 클러스터로 묶음. 각 클러스터는 `search_keywords`를 가짐.
3. **5채널 풀스캔**을 `reporter-review-scout` **5인스턴스 병렬**에
   위임 — 동일 clusters를 받아 각 채널이 **주제 키워드 기반 검색
   7종 패턴 강제**로 풀스캔.
4. **합성** — 5채널 yaml 병합 → 클러스터별 감성 분포·빈출 키워드·
   대표 인용·비판/칭찬 패턴. 기자명 직접 거론 게시글은 별도 boucket.
5. **산출** — 종합 리포트 1장 + 채널별 raw yaml 5개 + article_list 1개를
   `./specs/reporter-review-scan/<reporter-slug>/<today>/`에 저장.

### 메인/보조 구분

- **메인 산출**: 주제 클러스터별 여론 반응
- **보조 산출**: 기자 개인 평판 (기자명 직접 거론 0건이면 정직히 0건 기재)

### 왜 v2인가

v1 운영 결과(yu-seolhee 케이스, 2026-05-24): 기자명을 직접 거론한
게시글·댓글이 5채널 전수에서 **0건** — 이는 도구 한계가 아니라 **한국
뉴스 소비 패턴의 구조적 사실**. 독자는 「기사 주제」에 반응하지
「기자 이름」에 반응하지 않는다. 따라서 메인 검색축을 주제 키워드로
전환했고, 기자 평판은 보조로 편입했다.

## 언제 트리거되는가

- "<언론사> <기자> 보도 반응/평판/리뷰 봐줘·모아줘"
- "<기자>가 다룬 이슈·주제 여론"
- "<기자> 기사 주제별 평가"
- "<언론사> <기자> 최근 N개월 반응 정리"
- "/reporter-review-scan <언론사> <기자> [기간]"

## 트리거되지 않는 경우

- 언론사 전체 평판(특정 기자 아님) → 별도 하네스 필요(현재 미구현)
- 특정 기사 1개 반응만 → 일반 WebSearch로 충분
- 기자 신상 조사·실명 검증 → **거부.** 이 하네스는 보도 주제 여론
  수집까지가 본분, 신상 캐기 도구 아님.
- 특정 사건 1개의 여론 분석 → 일반 WebSearch 또는 별도 하네스
- 코드베이스 분석 → `/reverse`

## 어떻게 동작하는가

이 스킬이 트리거되면 **`/reporter-review-scan` 커맨드의 실행 순서를
그대로 수행**한다:

1. **Step 0** — 입력 파싱(언론사·기자·기간), 메타·저장 경로 결정.
   언론사/기자가 비면 한 번만 묻고, 그 외엔 끝까지.
2. **Step 1** — 기사 목록 수집을 `reporter-article-collector`에
   위임. summary·keywords까지 받음.
3. **Step 2** *(신규)* — 오케스트레이터 본체가 **주제 클러스터링**
   수행 (3~6개 클러스터, cluster_id·search_keywords 부여).
   → `01_article_list.yaml` 저장(클러스터 매핑 포함).
4. **Step 3** — `reporter-review-scout` **5인스턴스 동시 호출**
   (단일 메시지에 5 Task 병렬). 각각 자기 채널만 **7종 쿼리 패턴
   강제로 풀스캔** → `02~06_reviews_*.yaml` 5개 저장.
5. **Step 4** — 5채널 합성: 클러스터별 여론(메인) + 기자명 거론
   bucket(보조) → `00_report.md` 저장.
6. **Step 5** — grep 결정적 자가검증(파일·키·yaml 6개·출처·
   메인/보조 섹션) → 작업 로그 append → 요약 보고.

상세 계약(Contract·불변식)은 `.claude/commands/reporter-review-scan.md`
를 따른다. **평판 단정·명예훼손·실명 추정은 절대 금지**, 기자명
거론 0건을 우회 표현으로 호도 금지 — 사실 종합·패턴 식별까지가
본분이며, 평판 해석 최종 책임은 사용자에게 있다.
