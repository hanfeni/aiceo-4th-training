# consult-agent — 에이전트 요청 컨설팅 하네스

## 무엇을 하는가

수강생이 제출한 **LLM 에이전트 요청 사항 텍스트**를 받아,
강사가 컨설팅할 때 필요한 4가지 산출물을 자동 생성한다:

1. **에이전트 구조 다이어그램** (ASCII)
2. **필요 도구/API 목록** (기본 내장 vs 별도 권한 필요)
3. **페인 포인트 분석** (기술적 난관 + 현실적 제약)
4. **대안/우회 방법 + 구현 난이도** (상/중/하)

결과물은 `docs/consulting/<requestor>/<slug>.md`에 저장된다.

## 언제 트리거되는가

- `/consult-agent <요청 내용>` 커맨드 호출
- "이 에이전트 컨설팅해줘", "에이전트 분석해줘"
- "요청 사항 검토해줘", "수강생 에이전트 분석"

## 트리거되지 않는 경우

- 새 에이전트 직접 구현 요청 → `/implement`
- 코드베이스 분석 → `/reverse`
- 일반 질문 → 그냥 대화

## 어떻게 동작하는가

이 스킬이 트리거되면 `/consult-agent` 커맨드의 실행 순서를 그대로 수행한다:

1. **Step 0** — 요청자/슬러그/저장 경로 결정
2. **Step 1** — 에이전트 구조 파악 및 ASCII 다이어그램 생성
3. **Step 2** — 도구/API 목록 분류 (내장 vs 별도 권한)
4. **Step 3** — 페인 포인트 분석 (기술적 + 현실적 제약)
5. **Step 4** — 대안/우회 방법 및 난이도 평가
6. **Step 5** — 마크다운 저장 + 자가검증 + 보고

요청 내용이 없으면 한 번만 물어보고, 채워지면 질문 없이 끝까지.
상세 계약은 `.claude/commands/consult-agent.md`를 따른다.
