# korea-employment-legal — 한국형 사내 노무 법무 점검 진입점

## 무엇을 하는가

사용자가 **점검 대상(근로계약서/취업규칙/인사정책) 또는 분쟁
사실관계**를 주면, 오케스트레이터가 **단일 워크플로우로 직접
점검**(오리지널 plugin 토폴로지 — subagent 없음)해 ① 고용 유형·
쟁점 파악 ② 근로기준법·노동조합법·남녀고용평등법 등 점검
③ 노동위·고용노동부 기준 대조 ④ 분쟁 리스크 평가 ⑤ 권고를
수행하고, 점검 또는 분쟁 대응 자료 마크다운 1개를
`./specs/claude-for-x-kr/korea-employment-legal/<slug>/<today>/`
에 저장한다.

이것은 Anthropic Legal의 **Employment Legal** plugin을 기능·
토폴로지·출력 불변으로 한국에 이식한 것이다. 변형은 **3개
레이어**뿐이다 — 데이터(FLSA/Title VII·美 판례 → 근로기준법·
노동위), 법체계(at-will 고용 → 한국 해고제한·정당한 이유 법리),
법조분류(美 employment practice → 한국 노동법 실무).

## 언제 트리거되는가

- "근로계약서/취업규칙/인사정책 근로기준법 위반 점검"
- "권고사직 합의서 검토", "부당해고/임금체불 대응 자료"
- "직장내괴롭힘 대응 절차", "노동위 사건 대응 정리"

## 트리거되지 않는 경우

- 일반 상거래 계약 → `/korea-commercial-legal`
- 개인정보 처리방침·위탁 점검 → `/korea-privacy-legal`
- 자영업자 1차 계약 훑기 → `/korea-contract-reviewer`
- 본격 법률·노무 자문, 노동위 대리 → 변호사·공인노무사

## 어떻게 동작하는가

이 스킬이 트리거되면 **`/korea-employment-legal` 커맨드의 실행
순서를 그대로 수행**한다:

1. **Step 0** — 점검 대상·사실관계 수령(비면 한 번만 요청),
   Mode(A 문서 점검 / B 분쟁 대응) 식별, 저장 경로 결정.
   근로자 식별정보는 산출물·로그에 마스킹.
2. **Step 1** — 오케스트레이터가 단일 워크플로우로 직접 점검
   (쟁점 파악 → 근기법·노조법 등 점검 → 노동위·고용부 기준
   대조 → 리스크 평가 → 권고). subagent 없음(오리지널 plugin
   토폴로지 보존).
3. **Step 2** — 마크다운 1개 저장 → grep 결정적 자가검증(변호사법
   §109 + 공인노무사법 + 전문가 검토 디스클레이머 + 인용 실재
   검증 포함) → 로그 append → 요약 보고.

대상이 비면 한 번만 묻고, 그 외엔 질문 없이 끝까지. 상세 계약은
`.claude/commands/korea-employment-legal.md`를 따른다. 변호사·
공인노무사 자문을 대체하지 않으며, 중대 사안은 "변호사·노무사
검토 권장"을 강제 표시한다.
