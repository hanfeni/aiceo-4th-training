# /korea-content-strategist — 한국형 콘텐츠 전략 하네스

## 0. 정체성 (오리지널 패턴 충실 — 무엇이 불변인가)

이 하네스는 Anthropic Small Business 라인업의 **Content Strategist**
(토폴로지 = **Skill**, 단일 워크플로우, 명시 커넥터·subagent 없음)를
**기능·토폴로지·출력 포맷 불변**으로 한국에 이식한 것이다.

**오리지널에서 변하지 않는 것 (절대 변형 금지):**

- **기능**: 마케팅 메시지 **기획**(전략 수립). 콘텐츠를 대량 생성
  하는 게 아니라, 무엇을·누구에게·어디서·어떤 앵글로 말할지의
  **전략·메시지 설계**.
- **토폴로지**: **Skill 단일 워크플로우.** 오리지널 명세가 단촐
  하다(커넥터·subagent 없음). 단촐함을 우리 패턴(합성·중복 병렬
  등)으로 부풀리지 않는다 — 그게 pattern bleed다. agent 파일을
  만들지 않는다(command+SKILL 2파일).
- **출력**: 콘텐츠 전략 1개(메시지 기둥 → 타깃·톤 → 채널별 앵글
  → 콘텐츠 아이디어 → 실행 캘린더 골격).

**변형되는 것 — 사실상 문화 레이어 하나:**

| 레이어 | US 오리지널 | → 한국 |
|---|---|---|
| 문화·채널 | 美 마케팅 카피·메시지 관행, 美 SNS·이메일 채널 관행 | 한국 콘텐츠·카피 관행(존댓말·정서·후킹 어조), 한국 채널 생태계(네이버 블로그·카카오·인스타·유튜브 숏폼·당근 등), 한국 소비자 정서 |

(오리지널에 데이터 커넥터가 없으므로 데이터 소스 치환은 거의
없다. 필요 시 WebSearch로 한국 채널 트렌드를 참고하되, 그건
보조이지 새 토폴로지가 아니다.)

**이 하네스가 하지 않는 것:** 광고비 집행·계정 운영 대행, 허위·
과장 카피 생성, 규제 분야(의료·금융·건강기능식품 등) 표현 가이드
없는 카피 단정. 규제 분야면 "해당 분야 광고 규제 별도 확인 필요"
를 명시한다.

## 1. Step 0 — 입력 수령 + 저장 경로 (오케스트레이터 본체)

입력은 `$ARGUMENTS`(사업/제품 + 타깃 + 목표). **비어 있으면
"무엇을, 누구에게, 어떤 목표로 알리고 싶은지"를 한 번만 요청**하고,
채워져 있으면 질문 없이 끝까지.

```bash
now_kst=$(TZ=Asia/Seoul date '+%Y-%m-%d %H:%M KST')
today=$(TZ=Asia/Seoul date '+%Y-%m-%d')
run_id=$(TZ=Asia/Seoul date '+%H%M%S')
```

- `brief` — 정규화한 한 줄(사업·타깃·목표). 오케스트레이터가 한 번만.
- `slug` — kebab-case(예: `local-bakery-revisit-30s`).

저장 경로(프로젝트 로컬):

```bash
base="./specs/claude-for-x-kr/korea-content-strategist/<slug>/<today>"
mkdir -p "$base"
if [ -f "$base/00_content_strategy.md" ]; then
  out="$base/00_content_strategy_<run_id>.md"
else
  out="$base/00_content_strategy.md"
fi
```

## 2. Step 1 — 콘텐츠 전략 설계 (단일 워크플로우 — 오케스트레이터 직접)

subagent 없이 오케스트레이터가 직접 설계한다(오리지널 Skill 단일
워크플로우 보존). 필요 시 WebSearch로 한국 채널 트렌드·관행을
보조 참고(보조이지 토폴로지 변경 아님).

설계 절차:

1. **타깃·인사이트 정의** — 누구에게 말하는가, 그들의 맥락·페인·
   동기. 입력이 모호하면 합리적 가정을 **가정으로 명시**(지어낸
   사실로 단정하지 않는다).
2. **메시지 기둥(message pillars) 3~5개** — 이 사업이 일관되게
   말할 핵심 메시지 축. 각 기둥의 한 줄 정의.
3. **톤·보이스** — 한국 정서·존댓말·후킹 어조 관행에 맞춘 톤 가이드
   (예: 친근/전문/위트). 한국 소비자 정서 고려.
4. **채널별 앵글** — 한국 채널 생태계별로 같은 메시지를 어떻게
   변주할지: 네이버 블로그(정보형 long), 인스타(비주얼·짧은 후킹),
   유튜브/릴스 숏폼, 카카오 채널(알림·재방문), 당근·지역 커뮤니티
   등. 사업 성격에 맞는 채널만 선택(전 채널 나열 강요 아님).
5. **콘텐츠 아이디어** — 기둥×채널 교차로 즉시 쓸 수 있는 아이디어
   목록(제목/앵글 수준. 완성 카피 대량 생성 아님 — 전략이 본분).
6. **실행 캘린더 골격** — 4주 정도의 발행 리듬 예시(빈도·기둥 배분).

규제 주의: 의료·금융·건강기능식품 등 **규제 분야면 "표현 규제
별도 확인 필요"를 해당 섹션에 명시**하고 단정 카피를 만들지 않는다.

환각 방어(엄수): 입력에 **없는 사실(매출·고객수·수상 등)을 지어
넣지 않는다.** 가정은 "가정:"으로 표시. WebSearch 참고 시 출처
표기. 트렌드를 단정하지 말고 "최근 ~경향(출처)" 형태로.

## 3. Step 2 — 마크다운 1개 저장 + 결정적 자가검증 + 로그

§4 Contract대로 **마크다운 1개**를 `$out`에 Write.

결정적 자가검증(기계 확인 후에만 완료 보고):

```bash
test -f "$out" && echo "FILE_OK $out" || echo "FILE_MISSING $out"
for k in harness brief slug run_at run_id pillar_count channel_count; do
  grep -q "^$k:" "$out" && echo "key $k OK" || echo "key $k MISSING"
done
# 메시지 기둥 섹션이 실제로 존재하는지 (오리지널 핵심 = 메시지 기획)
grep -q '메시지 기둥\|message pillar' "$out" && echo "PILLAR_OK" || echo "PILLAR_MISSING"
```

검증 통과(파일 + 키 + 메시지 기둥 섹션) **뒤에만** 완료 보고.
실패면 md 수정 후 재검증.

로그 1줄 append:

```bash
log=".claude/history/daily/<today>.md"
[ -f "$log" ] || printf '## Claude Code 작업 로그\n\n' > "$log"
# - HH:MM | [<device-label>] | korea-content-strategist | <brief> — 기둥 N개, 채널 M개
```

device-label은 `$CLAUDE_DEVICE_LABEL` 또는
`~/.claude/hooks/get-device-label.sh` 출력.

사용자 보고: 저장 경로, 메시지 기둥 수, 선택 채널 수, 핵심 전략
1~2문장.

## 4. 최종 md Contract (구조 고정 — 표현 자유)

frontmatter에 아래 키 **전부**. Contract 외 키 금지.

```yaml
---
harness: korea-content-strategist
brief: <정규화 한 줄>
slug: <slug>
run_at: <now_kst>
run_id: <run_id>
pillar_count: <메시지 기둥 수>
channel_count: <선택 채널 수>
---
```

본문 섹션(모두 포함, 요약 맨 앞):

1. **요약** *(맨 앞)* — 누구에게·무엇을·어떤 톤으로의 한눈 전략.
2. **타깃·인사이트** — 타깃 정의, 맥락·페인·동기. 가정은 "가정:"
   표시.
3. **메시지 기둥** — 3~5개 기둥, 각 한 줄 정의.
4. **톤·보이스** — 한국 정서·관행 반영 톤 가이드.
5. **채널별 앵글** — 선택 채널별 메시지 변주(사업에 맞는 채널만).
6. **콘텐츠 아이디어** — 기둥×채널 교차 아이디어 목록.
7. **실행 캘린더 골격** — 4주 발행 리듬 예시.
8. **주의·한계** — 규제 분야 표현 확인 필요(해당 시), 가정 기반
   범위, WebSearch 참고 출처.

---

## 5. 불변식 (어기면 하네스 실패)

1. 오리지널 토폴로지 보존 — Skill 단일 워크플로우. 단촐함을 우리
   패턴(합성·중복 병렬)으로 부풀리지 않는다(agent 파일 없음).
2. 기능은 메시지 **기획·전략**. 완성 카피 대량 생성으로 변질 금지.
3. 입력에 없는 사실 생성 금지. 가정은 "가정:" 표시. 트렌드는
   인용 형태.
4. 규제 분야(의료·금융·건기식 등)는 "표현 규제 별도 확인 필요"
   명시, 단정 카피 금지.
5. 저장 후 grep 기계 확인한 뒤에만 완료 보고.
