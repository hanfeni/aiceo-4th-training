# /korea-employment-legal — 한국형 사내 노무 법무 점검 하네스

## 0. 정체성 (오리지널 패턴 충실 — 무엇이 불변인가)

이 하네스는 Anthropic Legal 라인업의 **Employment Legal**
(토폴로지 = **Practice-Area Plugin**, setup interview로 시작,
단일 워크플로우, subagent 없음)를 **기능·토폴로지·출력 포맷
불변**으로 한국에 이식한 것이다.

**오리지널에서 변하지 않는 것 (절대 변형 금지):**

- **기능**: 고용 계약, 근로 분쟁, 인사 매뉴얼·정책 문서 점검.
  사내 인사·법무팀의 실무 점검(외부 의뢰인 법률서비스 아님).
- **토폴로지**: **Practice-Area Plugin 단일 워크플로우.**
  setup interview는 입력 수집 단계이지 subagent가 아니다.
  분업 위임 패턴을 끼우면 패턴 과잉 — agent 파일 없음
  (command+SKILL 2파일).
- **입력**: 점검 대상 문서(근로계약서/취업규칙/인사정책) 또는
  분쟁 사실관계 + (선택) 사내 인사정책.
- **출력**: 점검 리포트(요약 → 근기법 점검 → 대조 → 리스크 →
  권고) 또는 분쟁 대응 자료.

**변형되는 것 — 3개 레이어 (Legal은 3축):**

| 레이어 | US 오리지널 | → 한국 |
|---|---|---|
| 데이터 | FLSA·Title VII·美 판례 | 국가법령정보센터(근로기준법·노동조합법·남녀고용평등법·기간제법), 고용노동부 행정해석, 노동위원회 판정례 |
| 법체계 | at-will 고용 | 한국 해고 제한·정당한 이유 법리(근기법 §23) |
| 법조분류 | 美 employment practice | 한국 노동법·노무 실무 |

**setup interview 처리:** 사용자가 사내 인사정책을 함께
붙여넣으면 반영, 없으면 근기법·고용노동부 표준취업규칙·표준
근로계약서를 기본 기준으로 사용(메커니즘 보존, 학습 데이터만
한국화). "정책 미제공 시 표준 기준 적용" 정직 명시.

**이 하네스가 하지 않는 것:** 변호사·공인노무사 자문 대체,
노동위원회 사건 대리, 외부 의뢰인 법률서비스(변호사법 §109·
공인노무사법). 사내 1차 점검까지가 본분이며, 중대 사안은
"변호사·노무사 검토 권장" 강제 표시.

## 1. Step 0 — 입력 수령 + 저장 경로 (오케스트레이터 본체)

점검 대상/사실관계(+선택 인사정책)는 `$ARGUMENTS`로 받는다.
**비어 있으면 "점검 대상(근로계약서/취업규칙/인사정책) 또는
분쟁 사실관계를 붙여넣어 주세요"라고 한 번만 요청**하고, 채워져
있으면 질문 없이 끝까지.

```bash
now_kst=$(TZ=Asia/Seoul date '+%Y-%m-%d %H:%M KST')
today=$(TZ=Asia/Seoul date '+%Y-%m-%d')
run_id=$(TZ=Asia/Seoul date '+%H%M%S')
```

- `doc_type` — 입력에서 식별(근로계약서/취업규칙/인사정책/
  분쟁 사실관계). 불명확 시 "유형 불명확".
- `mode` — `A`(문서 점검, 기본) 또는 `B`(분쟁 대응. 발화에
  "부당해고/임금체불/괴롭힘/노동위" 있으면 B).
- `slug` — `doc_type` + 핵심 쟁점 kebab-case
  (예: `employment-contract-parttime-2026`). 근로자 식별정보
  (실명·주민번호·연락처)는 slug·로그·요약에 **쓰지 않는다**
  (원문에만 두고 산출물엔 마스킹).

저장 경로(프로젝트 로컬):

```bash
base="./specs/claude-for-x-kr/korea-employment-legal/<slug>/<today>"
mkdir -p "$base"
if [ -f "$base/00_employment_review.md" ]; then
  out="$base/00_employment_review_<run_id>.md"
else
  out="$base/00_employment_review.md"
fi
```

## 2. Step 1 — 점검 (단일 워크플로우 — 오케스트레이터가 직접)

subagent 없이 오케스트레이터가 직접 점검한다(오리지널 plugin
단일 워크플로우 보존). WebSearch/WebFetch로 **한국 현행 법령·
고용노동부 행정해석·노동위 판정례**를 대조한다(공개 자료만).

점검 절차:

1. **고용 유형·쟁점 파악** — 정규/기간제/단시간, 핵심 쟁점
   (임금·근로시간·해고·휴가·괴롭힘 등), 5인 미만 여부(적용
   특례 분기).
2. **근기법·노조법·평등법 등 점검** — 근로조건 명시(§17),
   해고 정당성·절차(§23·§27), 임금·퇴직금·연차, 직장내괴롭힘
   (§76의2·3), 차별. 5인 미만 사업장 적용 예외 분기.
3. **노동위·고용부 기준 대조** — 고용노동부 행정해석, 노동위
   판정례, 표준취업규칙·표준근로계약서. 대조 근거(조문/해석/
   판정례) 표기.
4. **분쟁 리스크 평가** — 위반 소지·분쟁 가능성 등급.
5. **권고 정리** — 조항·절차 보완 방향. Mode B면 분쟁 대응
   자료(쟁점 정리·근거·대응 절차). 중대 사안엔 "변호사·노무사
   검토 권장" 강제 표시.

환각 방어(엄수): **입력에 없는 사실·조항을 있다고 하지
않는다.** 법령 조문·노동위 판정례·행정해석을 인용할 땐
**WebFetch로 실재 확인된 것만 단정 인용**, 확인 못 한 것은
"확인 필요"로 표기하고 결정 근거에서 배제(지어내지 않는다).
법령 해석은 "~소지/가능성"으로 기술하고 변호사·노무사 검토
권고. 5인 미만 적용 특례를 임의 단정하지 말고 사업장 규모를
확인 요청.

## 3. Step 2 — 마크다운 1개 저장 + 결정적 자가검증 + 로그

§4 Contract대로 **마크다운 1개**를 `$out`에 Write.

결정적 자가검증(기계 확인 후에만 완료 보고):

```bash
test -f "$out" && echo "FILE_OK $out" || echo "FILE_MISSING $out"
for k in harness doc_type mode slug run_at run_id risk_item_count finding_count; do
  grep -q "^$k:" "$out" && echo "key $k OK" || echo "key $k MISSING"
done
# 변호사법 §109 + 공인노무사법 + 전문가 검토 디스클레이머
grep -q '변호사\|노무사' "$out" && echo "DISCLAIMER_OK" || echo "DISCLAIMER_MISSING"
grep -q '109\|공인노무사법\|법률자문이 아\|자문을 대체하지' "$out" && echo "BAR_ACT_OK" || echo "BAR_ACT_MISSING"
# 5인 미만 특례 인지 (오리지널 정직 이식 검증 — 한국 노동법 특유)
grep -q '5인 미만\|적용 특례\|사업장 규모' "$out" && echo "SCALE_NOTE_OK" || echo "SCALE_NOTE_MISSING"
# 인용 실재 검증 표기
grep -q '확인 필요\|소지\|가능성' "$out" && echo "CITATION_GUARD_OK" || echo "CITATION_GUARD_MISSING"
```

검증 통과 **뒤에만** 완료 보고. 실패면 md 수정 후 재검증.

로그 1줄 append (근로자 식별정보 마스킹 — 문서 유형만):

```bash
log=".claude/history/daily/<today>.md"
[ -f "$log" ] || printf '## Claude Code 작업 로그\n\n' > "$log"
# - HH:MM | [<device-label>] | korea-employment-legal | <doc_type> mode=<A|B> — 위험 N건, 발견 M건
```

device-label은 `$CLAUDE_DEVICE_LABEL` 또는
`~/.claude/hooks/get-device-label.sh` 출력.

사용자 보고: 저장 경로, 위험 항목 수, 발견 수, 핵심 권고
1~2문장, 변호사·노무사 검토 권고 명시.

## 4. 최종 md Contract (구조 고정 — 표현 자유)

frontmatter에 아래 키 **전부**. Contract 외 키 금지.

```yaml
---
harness: korea-employment-legal
doc_type: <식별한 문서 유형>
mode: <A | B>
slug: <slug>
run_at: <now_kst>
run_id: <run_id>
risk_item_count: <위험 항목 수>
finding_count: <발견 수>
---
```

본문 섹션(모두 포함, 요약 맨 앞):

1. **요약** *(맨 앞)* — 고용 유형·쟁점, 핵심 위험 한눈,
   변호사·노무사 검토 필요 여부, 5인 미만 특례 적용 여부,
   정책 적용/미적용 명시.
2. **근기법·노동법 점검** — 항목별: 현황 요지 / 위반·미흡
   소지 / 법 조문 근거(실재 확인분만, 미확인은 "확인 필요") /
   권고. (5인 미만 적용 예외 분기 명시)
3. **노동위·고용부 기준 대조** — 행정해석·판정례·표준양식
   대비 누락·이탈 + 보완 방향.
4. **분쟁 리스크 평가** — 항목별 등급(준수/미흡/고위험) + 사유.
5. **권고·대응** — 우선순위순. Mode B면 분쟁 대응 자료(쟁점
   정리·근거·대응 절차).
6. **한계·면책** — 1차 사내 점검임, **변호사·노무사 검토
   권장**(강제), 본 출력은 법률·노무 자문이 아니며 변호사법
   §109·공인노무사법 준수 사내 보조도구. 5인 미만 특례는
   사업장 규모 확인 전제. "확인 필요" 인용은 사용자 원문
   확인 요망.

## 5. 불변식 (어기면 하네스 실패)

1. 오리지널 토폴로지 보존 — Practice-Area Plugin 단일 워크
   플로우. subagent·분업 위임 없음(agent 파일 없음).
2. setup interview는 흉내내지 말고 "정책 미제공 시 표준 기준"
   정직 명시. 학습 데이터만 한국화.
3. 입력에 없는 사실·조항 생성 금지. 법령·판정례·행정해석은
   WebFetch 실재 확인분만 단정 인용, 미확인은 "확인 필요"로
   결정 배제. 5인 미만 특례 임의 단정 금지(규모 확인).
4. 중대 사안 "변호사·노무사 검토 권장" + 변호사법 §109 +
   공인노무사법 디스클레이머 강제. 노동위 대리·외부 의뢰인
   법률서비스로 포지셔닝 금지.
5. 근로자 식별정보(실명·주민번호·연락처)는 산출물·로그·
   slug에 마스킹.
6. 저장 후 grep 기계 확인한 뒤에만 완료 보고.
