# 2회차 실습 — 데이터 검색 도구 3단계 프롬프트

> 서울대 AI CEO 4기 **2회차(2026-05-22) "제작"** 실습 자산.
> 1회차 `bootstrap/` 5종과 같은 방식 — **코딩에이전트에 복붙하면
> 에이전트가 수행**한다. 직접 코드 짜지 않는다.

## 무엇인가

검색 문서에 ① 메타 스키마 발굴 → ② LLM 메타 라벨링 → ③ OpenSearch
하이브리드 검색을 구축하는 **완성 프롬프트 3개**. 사내 운영 시스템
(커뮤니티 AI검색)의 검증된 패턴을 강의 실습으로 이식한 것이다.

## 전제 — 데이터는 이미 있다

5개 도메인 실습 데이터가 `poc/data/<domain>/`에 수집 완료돼 있다
(강사 로컬, `.gitignore`로 git 제외 — 다른 노트북이면 별도 복사).

| 도메인 | dir | 구조화(SQL/그래프) | 검색 문서(임베딩) |
|---|---|---|---|
| 상권 | `sangkwon` | 상가정보 1만행 | 정책뉴스 300건 |
| 의료 | `medical` | 약가마스터 2만행 | 의약품 허가상세 300건 |
| 금융 | `finance` | 국민연금 2만행 | 정책뉴스 246건 |
| 법률 | `legal` | 법령메타 5,425건 | 법령본문 449건 |
| 정책 | `policy` | 세출예산 1,640행 | 정책브리핑 943건 |

수강생은 **자기 직군에 가까운 도메인 1개**를 골라 프롬프트의
도메인 인자로 쓴다.

## 복붙 순서 (3가지 트랙)

### 트랙 A — 풀코스 (메타까지 강화된 검색)

```text
03-메타스키마발굴-프롬프트.md   → meta_schema.json 산출 (일회성)
02-LLM메타라벨링-프롬프트.md    → 문서에 메타 부착 (_labeled.jsonl)
01-검색인덱싱-프롬프트.md       → OpenSearch 하이브리드 (메타 부스팅)
```

### 트랙 B — 빠른 체험 (검색만)

```text
01-검색인덱싱-프롬프트.md  단독 → BM25+벡터+하이브리드 (메타 없이)
```

### 트랙 C — 메타 실습만

```text
03 → 02  (LLM이 데이터 구조를 발견·라벨링하는 것만 체감)
```

## 사용법 (1회차 §10과 동일 방식)

1. 자기 도메인 결정 (`sangkwon`/`medical`/`finance`/`legal`/`policy`)
2. 코딩에이전트(Claude Code 등)를 프로젝트 루트에서 연다
3. 원하는 트랙의 프롬프트 파일 **본문 전체를 복사**해 에이전트에 붙여넣는다
4. 도메인을 묻거나 명시 (예: "도메인은 medical")
5. 에이전트가 절대 규칙(§0)을 지키며 끝까지 수행 — 사람은 키 부재
   같은 진행 불가 상황만 응답

## ⚠️ 주의 (검증된 함정)

- **GitHub blob 링크를 WebFetch로 읽히지 마라** — 영어 요약·변형됨.
  raw URL(`raw.githubusercontent.com`)로 원문을 받게 하라
  (1회차 README §10-C와 동일 함정).
- **OpenAI 키 필요** (임베딩·메타 라벨링). 키 없으면 프롬프트가
  멈추고 알리도록 설계됨 — 임의 진행 안 함.
- **데이터 재수집 금지** — 프롬프트가 막아두었으나, 에이전트가
  `poc/data/`를 못 찾으면 데이터부터 확보해야 한다(강의 노트북 이슈).
